[파일 저장 규칙]
* 모든 파일은 UTF-8 인코딩 사용
* 파일명은 영문, 하이픈(-) 사용, 공백 금지
* 폴더 구조는 계층적으로 구성
* 결과물은 항상 outputs/ 하위에 저장
* 파일명에 날짜 필수 : YYYY-MM-DD_파일명.md
* 기존 파일 덮어쓰기 금지, 버전 관리 필수

[작업 로그 규칙]
* 모든 작업은 작업 로그에 기록
* 작업 로그는 YYYY-MM-DD_work.log 형식으로 저장
* 로그 형식 : changes-YYYY-MM-DD.log
* 작업 로그는 outputs/logs/ 하위에 저장

[안전 규칙]
* 개인정보, 회사기밀 절대 유출 금지
* 인터넷 검색 시 보안 설정 준수
* 의심스러운 파일 실행 금지
* 보안 프로그램 항상 최신 상태 유지
* 파일 삭제 전 반드시 확인 요청
* projects/ 폴더의 원본 데이터는 읽기만 (수정/삭제 금지)
