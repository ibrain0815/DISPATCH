# PhotoPick AI — 개발 기획서

> 1000장+ 사진에서 인스타 베스트샷을 자동 선별하는 순수 클라이언트 웹 앱

---

## 1. 개발 목표

| 항목 | 내용 |
|---|---|
| 입력 | 폴더 드래그 앤 드롭으로 사진 대량 업로드 (최대 5000장) |
| 처리 | 3단계 AI 파이프라인 (필터 → 중복 제거 → 정밀 분석) |
| 출력 | S/A/B/C/D 등급 추천 결과 + 인스타 최적 크롭 + ZIP 다운로드 |
| 특징 | 서버 없음 (모든 연산이 브라우저 내 Web Worker에서 실행) |

---

```

├── workers/           ← Web Worker (메인 스레드와 분리)
│   ├── WorkerPool.ts      코어 수 기반 Worker 풀 매니저

│
├── components/
│   ├── UploadZone.tsx     폴더 드래그 앤 드롭 (재귀 탐색)

│   ├── PhotoGrid.tsx      결과 그리드 (등급 필터, 무한스크롤)
│   ├── PhotoDetail.tsx    선택 사진 상세 (레이더 차트, 패널티 목록)
│   ├── CropPreview.tsx    인스타 비율별 크롭 미리보기
│   └── ScoreChart.tsx     Recharts 레이더 차트
│
└── engine/
    └── pipeline.ts        3단계 파이프라인 오케스트레이터
```

## 4. 분석 점수 가중치

| 항목 | 가중치 | 설명 |
|---|---|---|
| 화질 (블러+노출) | 30% | 가장 기본, 필터 단계에서 이미 1차 제거 |
| 표정 (눈+미소) | 25% | 눈 감음/반쯤 감음이 큰 감점 |
| 구도 (삼분법+헤드룸+기울기) | 20% | 삼분법 교차점 근접도 |
| 조명 | 15% | 하이라이트 클리핑, 그림자 위치 |
| 배경 | 10% | 복잡도, 피사체 분리도 |

### 등급 환산

| 등급 | 점수 | 비율 (예상) |
|---|---|---|
| S | 85~100 | 상위 5% |
| A | 70~84 | 상위 20% |
| B | 55~69 | 중간 30% |
| C | 40~54 | 하위 30% |
| D | 0~39 | 탈락 (1차 필터 통과했지만 낮은 점수) |

## 5. 핵심 기술 결정 사항

### Worker 통신 방식
- `ArrayBuffer` transferable로 파일 데이터 전송 (복사 없이 이동)
- Worker → 메인: 결과 객체 (JSON 직렬화 가능한 형태만)
- MediaPipe 모델: Worker 내에서 초기화, 이후 재사용

### 메모리 전략 (1000장 기준)
- 썸네일: 200×200 Object URL (표시 후 revoke 지연)
- 분석용 버퍼: 320×240 ImageData (분석 완료 후 즉시 해제)
- 정밀 분석용: 원본 비율 유지 640px (MediaPipe 권장 해상도)

### pHash 임계값
- 해밍 거리 ≤ 10 → 유사 사진 (같은 그룹)
- 추가로 EXIF 촬영 시간 ±30초 이내 → 버스트 그룹
- 그룹 내 sharpness 최고값이 대표 사진

## 6. 개발 환경 세팅 명령어

```bash
# 프로젝트 생성
npm create vite@latest photopick-ai -- --template react-ts
cd photopick-ai

# 의존성 설치
npm install zustand exifr fflate recharts
npm install @mediapipe/face_mesh @mediapipe/pose
npm install @tensorflow/tfjs @tensorflow-models/body-pix
npm install -D tailwindcss @tailwindcss/vite @types/node

# 개발 서버
npm run dev
```
