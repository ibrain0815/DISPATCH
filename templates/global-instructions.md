## 💾 파일 저장 위치 결정 가이드

| 콘텐츠 종류 | 저장 위치 | 예시 |
|-----------|---------|------|
| **코드 파일** | `projects/work/` | `.py`, `.js`, `.go` |
| **분석 결과** | `outputs/` | 배관설계 리서치, 보고서 |
| **데이터** | `projects/data/` | CSV, JSON, 데이터셋 |
| **보고서** | `outputs/reports/` | 정기 보고서, 분석 문서 |
| **계획/플랜** | `outputs/plans/` | 프로젝트 계획, 구성안 |
| **스크립트** | `projects/scripts/` | 자동화 스크립트 |
| **로그** | `outputs/logs/` | 작업 일지, 오류 로그 |
| **템플릿** | `templates/` | 반복 사용 서식 |

## ✨ 작업 프로세스

### Step 1: 작업 시작
```
1. 작업 목표를 context/working-rules.md에 기록
2. 필요한 템플릿을 templates/에서 선택
3. projects/work/ 폴더에서 작업 시작
```

### Step 2: 데이터 관리
```
1. 입력 데이터 → projects/data/
2. 처리 스크립트 → projects/scripts/
3. 프로젝트 문맥 → projects/content/
```

### Step 3: 결과 저장
```
1. 최종 결과물 → outputs/
2. 분석 보고서 → outputs/reports/
3. 계획/플랜 → outputs/plans/
4. 로그 기록 → outputs/logs/
5. 체크: 파일명, 폴더 위치, 형식 확인
```

## 🎓 코드 작성 스타일 가이드

### ✅ 권장 (Good)
```python
# 사용자 이름 추출
def extract_username(email):
    """이메일에서 @ 앞 부분을 사용자명으로 추출"""
    return email.split('@')[0]
```
