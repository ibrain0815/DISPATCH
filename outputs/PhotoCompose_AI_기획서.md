## 1. 프로젝트 개요

### 1.1 프로젝트 목표

사용자가 업로드한 인물 사진의 구도, 조명, 표정, 배경 등을 AI 기반으로 자동 분석하여 점수화하고, 여러 장의 사진 중 가장 잘 나온 사진을 추천해주는 웹 애플리케이션을 개발한다.

### 1.2 핵심 가치 제안

- 비전문가도 사진 구도의 좋고 나쁨을 객관적으로 파악 가능
- 대량의 사진 중 베스트샷을 빠르게 선별하여 시간 절약
- 구도 개선 포인트를 시각적으로 안내하여 사진 실력 향상
- SNS/블로그 업로드용 최적 사진 자동 선정

### 1.3 타겟 사용자

| 사용자 그룹 | 설명 | 주요 니즈 |
|---|---|---|
| 일반 사용자 | 스마트폰으로 사진 촬영하는 비전문가 | 잘 나온 사진 빠른 선별 |
| 블로거/인플루언서 | SNS 콘텐츠 제작자 | 품질 높은 사진 선별 및 구도 검증 |
| 아마추어 사진가 | 사진 실력을 키우고 싶은 취미 활동가 | 구도 학습 및 개선 피드백 |

## 2. 핵심 알고리즘 설계

### 2.1 전체 분석 파이프라인

사진이 업로드되면 다음 5단계 파이프라인을 통해 종합 점수가 산출된다:

| 단계 | 모듈 | 설명 | 가중치 |
|---|---|---|---|
| 1 | 인물 검출 | MediaPipe/YOLO로 얼굴위치와 신체 키포인트 추출 | 25% |
| 2 | 구도 분석 | 삼분법, 황금비율, 시선 방향 등 구도 규칙 평가 | 30% |
| 3 | 조명/화질 분석 | 노출, 샤프니스, 색온도, 히스토그램 분석 | 20% |
| 4 | 표정/감정 분석 | 얼굴 표정 자연스러움, 눈 뜨임 여부 평가 | 15% |
| 5 | 배경 분석 | 배경 복잡도, 주체-배경 분리도 평가 | 10% |

**최종 점수** = Σ(각 모듈 점수 × 가중치) → 0~100점 스케일로 출력

### 2.2 모듈별 상세 알고리즘

#### 2.2.1 인물 검출 모듈

**사용 기술:** MediaPipe Face Mesh + BlazePose

얼굴의 468개 랜드마크와 신체 33개 키포인트를 실시간으로 추출하여 인물의 위치, 크기, 자세를 파악한다.

**추출 데이터:**
- 얼굴 바운딩 박스 (x, y, w, h)
- 얼굴 중심점 좌표 및 사진 내 비율
- 신체 키포인트 (33개점)
- 얼굴 방향 (yaw, pitch, roll)
- 다인원 사진의 경우 인물별 개별 검출

**평가 기준:**
- 얼굴 크기가 전체 사진의 15~40% 차지 → 최적
- 얼굴 방향이 정면 또는 3/4측면 → 가산점
- 여러 인물일 경우 인물간 균형 배치 평가

#### 2.2.2 구도 분석 모듈 (핵심)

이 모듈이 전체 점수의 30%를 차지하는 핵심 모듈이다. 다음 6가지 구도 규칙을 점수화한다:

| 구도 규칙 | 알고리즘 | 점수 범위 |
|---|---|---|
| 삼분법 (Rule of Thirds) | 인물 중심점과 4개 교차점 간 최소 거리 계산 | 0~100 |
| 황금비율 (φ Ratio) | 인물 위치가 1:1.618 비율 라인에 근접한 정도 | 0~100 |
| 시선 방향 (Gaze Direction) | 시선 방향 앞에 여백 공간이 충분한지 판단 | 0~100 |
| 헤드룸 (Headroom) | 인물 위 여백이 적절한지 평가 (5~15% 최적) | 0~100 |
| 중심 균형 (Center Balance) | 주체 무게중심과 프레임 중심 간 균형 | 0~100 |
| 기울기 (Tilt Detection) | 수평선/수직선 기울어짐 감지 | 0~100 |

**삼분법 알고리즘 상세:**

1. 사진을 가로 3등분, 세로 3등분하여 4개의 교차점(Power Points) 산출
2. 인물 얼굴 중심점과 각 교차점간 유클리드 거리 계산
3. 최소 거리를 사진 대각선 길이로 정규화 (0~1)
4. 점수 = max(0, 100 - 정규화거리 × 200)

**시선 방향 알고리즘:**

1. MediaPipe 얼굴 랜드마크에서 yaw(좌우 회전) 값 추출
2. yaw > 0이면 오른쪽, < 0이면 왼쪽 시선으로 판단
3. 시선 방향 앞 여백 비율 계산 (40~60% 최적)
4. 점수 = 100 - |실제여백비율 - 50| × 2

#### 2.2.3 조명/화질 분석 모듈

| 분석 항목 | 방법 | 최적 범위 |
|---|---|---|
| 노출 (Exposure) | 히스토그램 평균값 및 표준편차 분석 | 평균 100~180, 표준편차 40~80 |
| 샤프니스 (Sharpness) | Laplacian 변환 분산값 측정 | 분산값 > 100 |
| 얼굴 조명 균일도 | 얼굴 영역 좌우 휘도 비대칭도 | 좌우 휘도차 < 30% |
| 색온도 (Color Temperature) | 평균 RGB 비율으로 White Balance 평가 | 자연스러운 색감 |

#### 2.2.4 표정/감정 분석 모듈

**사용 기술:** TensorFlow.js FaceMesh + 커스텀 분류모델

**평가 항목:**
- 눈 뜨임 여부 (EAR: Eye Aspect Ratio > 0.2 → 뜨임)
- 자연스러운 미소 (Mouth Aspect Ratio 및 측 근육 위치)
- 얼굴 비대칭도 점검 (좌우 랜드마크 대칭성)
- 표정 분류: 행복/중립/불편 → 행복 표정일수록 가산점

#### 2.2.5 배경 분석 모듈

**사용 기술:** BodyPix / Segment Anything (SAM) - 브라우저 경량화

**평가 항목:**
- 주체-배경 분리도 (Segmentation Confidence)
- 배경 복잡도 (Edge Density)
- 배경 보케 (Bokeh) 효과 감지
- 배경-주체 색상 대비 (Color Contrast)

## 3. 사진 추천 알고리즘

### 3.1 종합 점수 산출 공식

```
Total Score = (Composition × 0.30) + (Person Detection × 0.25) + (Lighting × 0.20) + (Expression × 0.15) + (Background × 0.10)
```

### 3.2 추천 로직

1. 업로드된 모든 사진에 대해 5개 모듈 분석 실행
2. 각 사진별 종합 점수 산출 (0~100)
3. 점수 기반 등급 분류: S(90+), A(80+), B(70+), C(60+), D(60미만)
4. 상위 3장을 추천 사진으로 선정
5. 각 사진별 개선 포인트 자동 생성

### 3.3 유사 사진 필터링

같은 장소/시간대에 찍은 유사 사진 그룹에서 베스트샷만 추려하여 중복 제거:

- EXIF 메타데이터 기반 시간 그룹핑 (±5분 이내)
- 퍼셉추얼 해시(pHash) 기반 유사도 비교 (Hamming Distance < 10)
- 그룹 내 최고 점수 사진 1장만 추천 목록에 포함

**처리 흐름:**

3. TensorFlow.js/MediaPipe로 브라우저 내 AI 분석 실행
4. 분석 결과 점수화 및 등급 분류
5. 결과 대시보드 표시 + 추천 사진 하이라이트
6. 선택적으로 서버에 결과 저장 (사용자 동의 시)

## 7. 핵심 기술 구현 가이드

### 7.1 삼분법 점수 계산 의사코드

```
function calculateRuleOfThirds(faceCenter, imageSize):
    // 4개 교차점 산출
    thirdX = imageSize.width / 3
    thirdY = imageSize.height / 3
    powerPoints = [(thirdX, thirdY), (thirdX*2, thirdY),
                   (thirdX, thirdY*2), (thirdX*2, thirdY*2)]
    // 최소 거리 계산 및 정규화
    diagonal = sqrt(width² + height²)
    minDist = min(distance(faceCenter, p) for p in powerPoints)
    normalized = minDist / diagonal
    return max(0, 100 - normalized * 200)
```

### 7.2 얼굴 조명 균일도 계산

```
function calculateFaceLighting(faceRegion):
    grayscale = convertToGrayscale(faceRegion)
    leftHalf = grayscale[:, :width/2]
    rightHalf = grayscale[:, width/2:]
    leftMean = mean(leftHalf)
    rightMean = mean(rightHalf)
    asymmetry = abs(leftMean - rightMean) / max(leftMean, rightMean)
    return max(0, 100 - asymmetry * 300)
```

```
function findDuplicateGroups(photos):
```
