# PhotoPick AI

## 1000장+ 대량 사진에서 인스타 베스트샷을 자동 선별하는 웹 앱 기획서

- **핵심 목적:** 대량의 사진(1000장+)을 빠르게 분석하여 인스타그램에 올릴 만한 사진만 자동으로 골라주는 도구
- **플랫폼:** 웹 앱 (브라우저 내 처리, 서버 업로드 불필요)
- **작성일:** 2026년 4월 6일

## 1. 프로젝트 정의

### 1.1 해결하려는 문제

> 사진을 1000장 이상 찍었는데, 이 중에서 인스타에 올릴 만한 사진을 고르려면 한 장씩 보면서 비교해야 하니까 시간이 너무 오래 걸린다.

**현재의 고통:**
- 1000장을 하나하나 보면서 고르면 최소 1~2시간 소요
- 비슷한 사진이 수십 장씩 있어서 비교하기 힘듦
- 어떤 사진이 객관적으로 잘 나왔는지 판단이 어려움
- 고른 사진이 인스타 비율에 안 맞아서 다시 크롭해야 함

**이 앱이 해결하는 방식:**
- 1000장 → 3분 이내 자동 분석 완료
- 유사 사진 자동 그룹핑 → 그룹별 베스트 1장 자동 선정
- 인스타 최적 비율(1:1, 4:5, 1.91:1) 자동 크롭 미리보기
- 최종 추천 10~30장만 확인하면 끝

### 1.2 핵심 지표

| 항목 | 목표 |
|---|---|
| 1000장 분석 시간 | **3분 이내** (Web Worker 병렬 처리) |
| 최종 추천 수 | 전체의 **3~5%** (1000장 → 30~50장) |
| 유사 사진 중복 제거율 | **80%+** 중복 제거 |
| 사용자 만족도 | 추천 사진 중 실제 선택률 **70%+** |

## 2. 사용자 워크플로우

### 2.1 전체 흐름 (3단계)

```
[1단계] 폴더째 드래그 앤 드롭
         ↓ (자동)
[2단계] AI 분석 + 중복 제거 + 순위 매기기  ← 사용자 대기 (3분)
         ↓ (자동)
[3단계] 추천 사진 확인 → 선택 → 인스타 크롭 다운로드
```

### 2.2 상세 시나리오

**Step 1 — 업로드 (10초)**
- 폴더를 통째로 드래그 앤 드롭 (또는 폴더 선택)
- JPG/PNG/HEIC 자동 인식, 비이미지 파일 자동 무시
- 1000장 이상도 한 번에 가능 (브라우저 메모리 관리)

**Step 2 — 자동 분석 (3분)**
- 실시간 진행률 표시: "247/1000장 분석 중... (예상 잔여 1분 42초)"
- 백그라운드 Web Worker 처리 → 브라우저 멈춤 없음
- 분석 완료 시 알림 (브라우저 Notification)

**Step 3 — 결과 확인 및 다운로드 (2분)**
- 추천 사진 그리드 보기 (점수 높은 순)
- 마음에 안 드는 사진 스와이프로 제거
- 선택한 사진 인스타 비율로 자동 크롭
- ZIP 다운로드 또는 개별 저장

## 3. 대량 처리 아키텍처

### 3.1 속도가 핵심인 이유

1000장을 1장당 3초씩 순차 처리하면 **50분**이 걸린다. 이를 3분으로 줄이려면 아키텍처 수준의 최적화가 필요하다.

### 3.2 3단계 파이프라인

```
┌─────────────────────────────────────────────────────┐
│  1차 필터 (초고속) — 0.05초/장 → 1000장 50초       │
│  · 블러 감지 (Laplacian 분산)                       │
│  · 노출 이상 감지 (히스토그램)                       │
│  · 얼굴 유무 빠른 체크                              │
│  → 명백한 실패작 40~50% 탈락                        │
├─────────────────────────────────────────────────────┤
│  2차 중복 제거 — 0.02초/장 → 500장 10초            │
│  · 퍼셉추얼 해시(pHash) 유사도 비교                  │
│  · EXIF 시간 기반 그룹핑                            │
│  → 유사 사진 그룹별 대표 1장만 통과                  │
├─────────────────────────────────────────────────────┤
│  3차 정밀 분석 — 0.5초/장 → 150~200장 90초          │
│  · 구도 분석 (삼분법, 시선방향, 헤드룸)              │
│  · 표정 분석 (눈뜨임, 미소)                         │
│  · 조명 품질                                        │
│  · 인스타 적합도 점수                               │
│  → 최종 추천 30~50장 선정                           │
└─────────────────────────────────────────────────────┘
          총 소요시간: 약 50 + 10 + 90 = 150초 (2.5분)
```

### 3.3 병렬 처리 설계

```
메인 스레드 (UI)
  ├── Web Worker 1: 1차 필터 (250장 담당)
  ├── Web Worker 2: 1차 필터 (250장 담당)
  ├── Web Worker 3: 1차 필터 (250장 담당)
  ├── Web Worker 4: 1차 필터 (250장 담당)
  └── 결과 수집 → 2차 중복제거 → 3차 정밀분석 분배
```

- **CPU 코어 수 자동 감지** (`navigator.hardwareConcurrency`)로 Worker 수 결정
- 이미지는 Worker에 `OffscreenCanvas`로 전달 (메인 스레드 블로킹 방지)
- 메모리 관리: 한 번에 50장씩 로드 → 분석 후 해제 → 다음 50장 로드

### 3.4 메모리 관리 전략

1000장 원본을 동시에 메모리에 올리면 브라우저가 크래시된다. 따라서:

| 단계 | 이미지 해상도 | 메모리/장 | 동시 로드 |
|---|---|---|---|
| 1차 필터 | 320×240 리사이즈 | ~0.2MB | 50장씩 |
| 2차 중복제거 | 32×32 해시용 | ~0.004MB | 전체 해시만 보관 |
| 3차 정밀분석 | 640×480 리사이즈 | ~0.9MB | 10장씩 |
| 결과 미리보기 | 800px 썸네일 | ~0.5MB | 30~50장 |
| 다운로드/크롭 | 원본 해상도 | 원본 크기 | 1장씩 |

최대 메모리 사용량: **약 150MB** (브라우저 안전 범위 내)

## 4. 분석 알고리즘

### 4.1 1차 필터 — 불량 사진 빠른 제거

> 목적: 명백히 실패한 사진을 0.05초 만에 걸러내서 정밀 분석 대상을 절반으로 줄인다.

**4.1.1 블러 감지 (흔들린 사진 제거)**

```
sharpness = variance(Laplacian(grayscale_image))
if sharpness < 50:     → 탈락 (심하게 흔들림)
if sharpness < 100:    → 경고 (약간 흔들림, 감점 후 통과)
```

- Laplacian 필터는 320×240 리사이즈 이미지에서 1ms 이내 처리 가능
- 임계값은 카메라 종류별 캘리브레이션 필요 (초기값 50)

**4.1.2 노출 이상 감지 (너무 밝거나 어두운 사진)**

```
histogram = computeHistogram(grayscale_image)
mean_brightness = mean(histogram)
if mean_brightness < 40:    → 탈락 (너무 어두움)
if mean_brightness > 230:   → 탈락 (너무 밝음/화이트아웃)
```

**4.1.3 얼굴 유무 빠른 체크**

```
faces = MediaPipe.detectFaces(image, minConfidence=0.5)
if faces.length == 0:   → 탈락 (인물 없음)
if max(face.size) < 0.03 * image.size:  → 탈락 (인물 너무 작음)
```

- 인물 사진 위주이므로 얼굴이 없는 사진은 바로 제외
- 얼굴이 전체 사진의 3% 미만으로 작으면 인물 사진으로 부적합

### 4.2 2차 필터 — 유사 사진 중복 제거

> 목적: 같은 포즈/장소에서 연속으로 찍은 사진 10장 → 베스트 1장만 남긴다.

**그룹핑 알고리즘:**

```
function groupSimilarPhotos(photos):
    // Step 1: EXIF 촬영시간 기반 시간 그룹 (±3초 이내 = 같은 그룹)
    timeGroups = groupByExifTime(photos, threshold=3sec)

    // Step 2: 시간 그룹 내에서 pHash 유사도 비교
    for group in timeGroups:
        hashes = [computePHash(resize(p, 32, 32)) for p in group]
        clusters = clusterByHammingDistance(hashes, threshold=12)

        // Step 3: 각 클러스터에서 1차 필터 점수가 가장 높은 1장 선택
        for cluster in clusters:
            keep photo with max(sharpness + exposure_score)
            discard rest
```

**왜 ±3초인가:**
- 연사 모드: 초당 10장 → 3초면 30장이 같은 그룹
- 포즈 변경 간격: 보통 5초 이상 → 다른 그룹으로 분리
- EXIF 없는 경우: 파일명 순서 + pHash만으로 그룹핑 fallback

### 4.3 3차 분석 — 인스타 적합도 정밀 평가

> 목적: 1,2차를 통과한 150~200장을 정밀 분석하여 인스타에 적합한 순서로 랭킹한다.

**종합 점수 공식 (인스타 최적화 가중치):**

```
Instagram Score = (구도 × 0.25) + (표정 × 0.25) + (화질 × 0.20) + (조명 × 0.15) + (배경 × 0.15)
```

일반 사진 평가와 달리, 인스타용으로 **표정 가중치를 구도와 동일하게 25%로 올렸다.** 인스타에서는 자연스러운 표정이 구도만큼 중요하기 때문이다.

**4.3.1 구도 분석 (25%)**

| 항목 | 가중치 | 알고리즘 |
|---|---|---|
| 삼분법 배치 | 35% | 얼굴 중심 ↔ 교차점 최소거리 |
| 시선 방향 여백 | 25% | 시선 앞 여백 비율 40~60% 최적 |
| 헤드룸 | 20% | 머리 위 여백 5~15% 최적 |
| 수평 기울기 | 10% | 어깨선/눈선 기울기 감지 |
| 중심 균형 | 10% | 인물 무게중심과 프레임 중심 거리 |

**4.3.2 표정 분석 (25%) — 인스타 특화**

| 항목 | 가중치 | 판단 기준 |
|---|---|---|
| 눈 뜨임 | 30% | EAR(Eye Aspect Ratio) > 0.21 → 뜨임 |
| 자연스러운 미소 | 30% | 입꼬리 상승 + 눈 주름(Duchenne smile) |
| 얼굴 각도 | 20% | 정면 또는 3/4 측면 (yaw 0~35°) |
| 눈 초점 | 20% | 동공 위치가 카메라 방향 (홍채 중심 분석) |

**감점 요소 (인스타 킬러):**
- 눈 감김 → **-40점** (인스타에 절대 올릴 수 없는 사진)
- 입 크게 벌림 (하품/재채기) → **-30점**
- 심한 이중턱 각도 → **-15점**
- 반눈 (한쪽 눈만 감김) → **-25점**

**4.3.3 화질 분석 (20%)**

| 항목 | 측정 | 점수 기준 |
|---|---|---|
| 샤프니스 | Laplacian 분산 | > 200: 100점, < 50: 0점 |
| 노이즈 | 고주파 에너지 비율 | 낮을수록 높은 점수 |
| 해상도 적합도 | 인스타 최소 1080px 기준 | 1080px+ : 만점, 미만: 감점 |

**4.3.4 조명 분석 (15%)**

| 항목 | 측정 | 점수 기준 |
|---|---|---|
| 얼굴 밝기 | 얼굴 영역 평균 밝기 | 120~200: 최적 |
| 좌우 균일도 | 얼굴 좌우 밝기차 | 차이 < 25%: 만점 |
| 역광 감지 | 배경 vs 인물 밝기 비율 | 배경이 2배+ 밝으면 감점 |

**4.3.5 배경 분석 (15%)**

| 항목 | 측정 | 점수 기준 |
|---|---|---|
| 배경 깔끔함 | Edge Density (엣지 밀도) | 낮을수록 높은 점수 |
| 주체 분리도 | BodyPix 세그멘테이션 신뢰도 | > 0.8: 만점 |
| 배경 보케 | 배경 vs 전경 샤프니스 차이 | 차이 클수록 가산점 |
| 불필요 요소 | 쓰레기통, 표지판 등 감지 | 있으면 감점 |

## 5. 인스타그램 최적화 기능

### 5.1 자동 크롭 시스템

인스타그램에 올릴 때 비율이 안 맞아서 다시 크롭하는 번거로움을 없앤다.

**지원 비율:**

| 비율 | 용도 | 해상도 |
|---|---|---|
| 1:1 (정사각형) | 피드 기본 | 1080×1080 |
| 4:5 (세로) | 피드 세로형 (가장 큰 면적 차지) | 1080×1350 |
| 1.91:1 (가로) | 피드 가로형 | 1080×566 |
| 9:16 (스토리/릴스) | 스토리 및 릴스 | 1080×1920 |

**스마트 크롭 알고리즘:**

```
function smartCrop(image, targetRatio, faceData):
    // 1. 얼굴 중심점을 기준으로 크롭 영역 설정
    cropCenter = faceData.center

    // 2. 머리 위 여백 확보 (헤드룸 10%)
    cropCenter.y -= faceData.height * 0.1

    // 3. 시선 방향에 여백 추가
    if faceData.gazeDirection == "left":
        cropCenter.x += image.width * 0.05  // 왼쪽에 여백
    else:
        cropCenter.x -= image.width * 0.05  // 오른쪽에 여백

    // 4. targetRatio에 맞춰 크롭 영역 계산
    cropRect = calculateCropRect(cropCenter, targetRatio, image.size)

    // 5. 크롭 영역이 이미지 밖으로 나가면 조정
    cropRect = clampToImageBounds(cropRect, image.size)

    return crop(image, cropRect)
```

### 5.2 배치 다운로드

- 선택한 사진을 원하는 비율로 일괄 크롭
- ZIP 파일로 한 번에 다운로드
- 파일명 자동 정리: `insta_001.jpg`, `insta_002.jpg`, ...
- 인스타 업로드 순서 추천 (색감/분위기 유사한 것끼리 묶기)

### 5.3 캐러셀(다중 사진) 추천

인스타 캐러셀 포스트(최대 10장)를 위한 조합 추천:

```
function recommendCarousel(rankedPhotos, maxSlides=10):
    selected = [rankedPhotos[0]]  // 1장은 최고점 사진

    for photo in rankedPhotos[1:]:
        // 이미 선택된 사진과 너무 유사하면 스킵
        if isTooSimilar(photo, selected):
            continue
        // 색감/분위기 다양성 확보
        if addsDiversity(photo, selected):
            selected.append(photo)
        if len(selected) >= maxSlides:
            break

    // 첫 번째 사진이 가장 중요 (피드 썸네일)
    return sortByImpact(selected)
```

┌──────────────────────────────────────────┐

```

**화면 2: 분석 중**
```
┌──────────────────────────────────────────┐

## 7. 기술 스택

### 7.1 프론트엔드 (전부 브라우저에서 실행)

| 역할 | 기술 | 선정 이유 |
|---|---|---|
| UI 프레임워크 | React 18 + TypeScript | 컴포넌트 재사용, 타입 안전성 |
| 스타일링 | Tailwind CSS | 빠른 UI 개발, 반응형 |
| 상태 관리 | Zustand | 가볍고 빠른 전역 상태 (1000장+ 데이터) |
| 이미지 처리 | OffscreenCanvas + Web Worker | 메인 스레드 블로킹 방지 |
| 얼굴 검출 | @mediapipe/face_mesh (WASM) | 브라우저 내 실시간 얼굴 분석 |
| 신체 검출 | @mediapipe/pose (WASM) | 자세/포즈 분석 |
| 배경 분리 | @tensorflow-models/body-pix | 경량 세그멘테이션 |
| EXIF 파싱 | exifr | 촬영시간/GPS/카메라정보 추출 |
| 이미지 해시 | phash-wasm | 퍼셉추얼 해시 (중복 감지) |
| ZIP 생성 | fflate | 경량 ZIP 압축 (브라우저) |
| 차트 | Recharts | 점수 시각화 |
| 빌드 | Vite | 빠른 HMR, 트리쉐이킹 |

### 7.2 백엔드 (최소한의 역할)

| 역할 | 기술 | 비고 |
|---|---|---|
| 호스팅 | Vercel / Netlify | 정적 사이트 배포 (무료) |
| CDN | Cloudflare | WASM 모델 파일 캐싱 |

**서버가 거의 필요 없는 구조:**
- 모든 AI 분석이 브라우저에서 실행됨
- 사진이 서버로 전송되지 않음 (프라이버시 보장)
- 서버 비용 = 거의 0원 (정적 호스팅만)

### 7.3 왜 브라우저 처리인가

| 비교 항목 | 서버 처리 | 브라우저 처리 (채택) |
|---|---|---|
| 1000장 업로드 | 네트워크 10분+ | 업로드 불필요 (0초) |
| 서버 비용 | GPU 서버 필요 (월 수십만원) | 무료 |
| 프라이버시 | 사진이 서버에 전송됨 | 사진이 기기를 떠나지 않음 |
| 속도 | 네트워크 병목 | 로컬 CPU/GPU 활용 |
| 오프라인 | 불가 | PWA로 오프라인 가능 |

## 9. 개발 로드맵 (12주)

기존 16주에서 **12주로 단축** — 인스타 선별이라는 명확한 목적에 집중하여 불필요한 기능을 제거했다.

### Phase 1: 빠른 필터링 엔진 (1~3주)

| 주차 | 작업 | 완료 기준 |
|---|---|---|
| 1주 | 프로젝트 세팅, 폴더 업로드 UI, Web Worker 기본 구조 | 1000장 폴더 드래그 앤 드롭 작동 |
| 2주 | 1차 필터 구현 (블러/노출/얼굴유무), 병렬 처리 | 1000장 1차 필터 60초 이내 |
| 3주 | 2차 중복 제거 (pHash + EXIF 시간 그룹핑) | 유사 사진 그룹핑 정확도 85%+ |

### Phase 2: 정밀 분석 + 추천 (4~6주)

| 주차 | 작업 | 완료 기준 |
|---|---|---|
| 4주 | 구도 분석 모듈 (삼분법, 시선방향, 헤드룸) | 구도 점수 산출 작동 |
| 5주 | 표정 분석 (눈뜨임, 미소, 감점요소) + 조명/배경 분석 | 5개 모듈 전체 작동 |
| 6주 | 종합 점수 산출, 등급 분류, 추천 로직 | 1000장 → 30~50장 추천 완료 |

### Phase 3: 인스타 최적화 UI (7~9주)

| 주차 | 작업 | 완료 기준 |
|---|---|---|
| 7주 | 추천 결과 그리드 UI, 등급 배지, 필터링 | 결과 화면 완성 |
| 8주 | 스마트 크롭 (1:1, 4:5, 9:16), 크롭 미리보기 | 인스타 비율 크롭 작동 |
| 9주 | 사진 상세 보기, 구도 오버레이, 비교 모드 | 상세 분석 UI 완성 |

### Phase 4: 최적화 + 배포 (10~12주)

| 주차 | 작업 | 완료 기준 |
|---|---|---|
| 10주 | ZIP 다운로드, 캐러셀 추천, 파일명 자동 정리 | 배치 다운로드 작동 |
| 11주 | 성능 최적화 (메모리, Worker, 모델 로딩) | 1000장 3분 목표 달성 |
| 12주 | PWA 설정, 반응형 UI, 테스트, Vercel 배포 | 서비스 런칭 |
